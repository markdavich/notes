# GitHub Enterprise - Packages

## Packages

Packages are **ready-made pieces of work** that teams can share and reuse —  
like supplies in a shared storage room.

### Core Ideas

|    Core Idea | What it is                      | Think of it like           |
| -----------: | :------------------------------ | :------------------------- |
|  **Package** | A reusable piece of work        | A toolbox you didn’t build |
| **Registry** | Where packages are stored       | A warehouse                |
|  **Version** | A specific edition of a package | The 2024 model of a car    |

### Why Packages Matter

|         Benefit | What it is                    | Think of it like        |
| --------------: | :---------------------------- | :---------------------- |
|       **Reuse** | Using something again         | Borrowing a ladder      |
| **Consistency** | Everyone using the same tools | Company-issued laptops  |
|       **Speed** | Getting started faster        | Pre-assembled furniture |

### Access & Safety

|         Concept | What it is               | Think of it like         |
| --------------: | :----------------------- | :----------------------- |
|  **Visibility** | Who can see a package    | Public vs private files  |
| **Permissions** | Who can use or change it | Keys to the storage room |

### Everyday Summary

Packages help teams **share, reuse, and stay organized** so everyone works  
faster and smarter.
